# Module Spec: The Robotic Nervous System (ROS 2)

**Module**: 01
**Topic**: ROS 2 Middleware
**Status**: Draft

## Objective
Introduce ROS 2 as the middleware foundation for humanoid robotics and physical AI systems.

## Learning Outcomes
- Understand nodes, topics, services, actions, and parameters.
- Create and manage ROS 2 workspaces and packages.
- Use ROS 2 CLI tools for debugging and introspection.
- Understand DDS-based communication.

## Chapter Outline
1. **ROS 2 Core Architecture**
2. **Environment Setup (Ubuntu + ROS 2)**
3. **Nodes and Executors**
4. **Topics, Services, and Actions**
5. **ROS 2 Package Management**
6. **Launch Files and Parameters**
7. **Debugging and Best Practices**

## Success Criteria
- Readers can run ROS 2 demo nodes.
- Readers can explain ROS 2 communication patterns.

## Constraints
- Linux-based environment only.

## Out of Scope
- Nav2, MoveIt2, advanced sensors.
